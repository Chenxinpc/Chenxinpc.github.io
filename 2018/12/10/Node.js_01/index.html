Node.js介绍

基于Chrome的V8 JS解析引擎之上，解放了Javascript的编程能力，为Javascript提供了后端编程能力；所以说，Nodejs是一个让Javascript进行后端编程的一个运行平台；
v8引擎：v8引擎是一个Javascript引擎实现，最初由一些语言专家设计，后被谷歌收购，随后谷歌对其进行了开源。v8使用C++开发，js在v8引擎下运行媲美二进制的速度（编译型语言和解析性语言）
编译型语言：linux java c++ c#
作用：可以在nodejs的平台上使用js语法来编写后台逻辑。

LTS和Current版本区别

1.LTS是长期稳定版的意思（这个安装包用起来比较稳定）【推荐在企业中使用】
2.Current是最新特征版，这个安装包中有最新的Node特征，但是，可能有一些潜藏的Bug未解决【推荐学习或尝鲜去使用】

环境变量

Path环境变量的作用：能让我们通过命令行的形式，快速启动一些应用程序；

系统环境变量和用户环境变量的区别

1.用户环境变量，是每一个用户私有的，用户之间不共享；
2.全局变量，是共享的，只要你能登陆这台计算机，就能访问到全局环境变量；【今后在配置环境变量的时候，推荐直接配置到系统环境变量】

CommonJS模块

定义来模块的语法

  • 1.什么是commonJS？
    commonjs就是为来了实现javascript模块化，而制定的一套规范
  • 2.为什么要使用commonJs
    为了同意大家写模块时的规范，方便各个模块之间的依赖和调用，所以commonJs就产生
  • 3.commonJS到底是什么东东
    • 定义了什么是模块
    • 定义了模块与模块之间如何去引用，如何去暴露 requore() exports()
    • 如果没有commonJS的规范，肯定是不行
    • 大家遵循这个规范，共同协作，方便他人方便自己
